# Hello-Wine-AppImage

This project demonstrates how to package and run a Windows `.exe` program within a self-contained Linux AppImage. It leverages a portable Wine AppImage to provide the necessary Windows environment.

## ✨ Features

*   **Portable Windows Execution:** Run Windows applications directly on Linux without complex installations.
*   **Self-Contained AppImage:** Distribute your Windows application as a single, easy-to-share executable.
*   **Pre-configured Wine Environment:** Uses a portable Wine AppImage and a pre-built Wine prefix for quick setup.

## 🚀 Getting Started

### 1. Build the AppImage

To build the `Hello-x86_64.AppImage` yourself, follow these steps:

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/myridia/Hello-Wine-AppImage.git
    cd Hello-Wine-AppImage
    ```

2.  **Make the AppImage tool executable:**
    ```bash
    chmod +x appimagetool-x86_64.AppImage
    ```

3.  **Build the main AppImage:**
    This command uses `appimagetool` to bundle your application (`hello.AppDir/`) into a final AppImage. We specify the architecture for the build.
    ```bash
    ARCH=x86_64 ./appimagetool-x86_64.AppImage hello.AppDir/
    ```
    This will create `Hello-x86_64.AppImage` in the current directory.

### 2. Run the AppImage

Once you have the `Hello-x86_64.AppImage`, you can run it:

1.  **Make the main AppImage executable:**
    ```bash
    sudo chmod +x Hello-x86_64.AppImage
    ```
    *(Note: `sudo` might be needed depending on your file system's permissions. Usually, `chmod +x` is sufficient.)*

2.  **Execute the AppImage:**
    ```bash
    ./Hello-x86_64.AppImage
    ```
    This will launch the Windows `.exe` program packaged inside.

## 💡 How It Works (Background)

This project uses a few key components to make Windows executables run portably on Linux:

*   **Portable Wine:** We rely on a self-contained Wine environment. The `wine-stable_11.0-x86_64.AppImage` (maintained by [mmtrt/WINE_AppImage](https://github.com/mmtrt/WINE_AppImage)) is used. This ensures Wine is available regardless of the host Linux system.
*   **Pre-built Wine Prefix:** A pre-configured Wine prefix is included (`hello.AppDir/prefix/`). This prefix contains necessary settings and potentially installed components for your Windows program.
*   **Exporting the Prefix:** Since AppImages typically mount with a read-only filesystem, we need to make the Wine prefix accessible to the user's home directory on first run. The AppImage runtime handles exporting this `prefix` to the user's `~/.wine` (or similar location) so that Windows applications can correctly access their configurations and settings.

## 🧪 Testing the Build Action

You can test the GitHub Actions workflow locally using `act`.

1.  Navigate to the workflows directory:
    ```bash
    cd .github/workflows/
    ```

2.  Run the workflow locally:
    ```bash
    act -W linux.yml --verbose -P docker3=myridia/runner --pull=false
    ```
    *(This command requires `act` to be installed and configured.)*
